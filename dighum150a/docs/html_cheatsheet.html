Here are some Useful HTML Tags

At the top of your HTML document, outside of other elements:

<!DOCTYPE html>

Every HTML document should have this at the top. It tells the browser or other interpreting software the format of the document that it is reading. Note that the ! means that the contents of the tag are not displayed or interpreted by the browser. It's just a message to the browser.

<html lang=“en” /></html>

The <html> tag has to wrap all of the content on your site including the head and body tags. The attribute "lang" tells your browser or screen-reader which language the page is in. This will be useful later when you start to make pages for multiple languages. You can look up codes for other languages here: https://www.loc.gov/standards/iso639-2/php/code_list.php

The Header

The head tag contains information about your page which gets displayed at the top of the browser on some computers.  This is where you can place metadata which will help archiving software, and search engines access your site, as well as anyone else who wants to scrape your site for their own projects. You can also place information here which helps your browser or screen-reader to interpret and display the content of your site, like the charset metadata.

<head></head>

This marks the header of the webpage, and you should nestle the following elements inside it.

<meta charset="utf-8” />

This meta element tells your browser to use unicode when displaying the characters on the site. Most pages use unicode these days, but you can read about the charsets of old here: https://www.iana.org/assignments/character-sets/character-sets.xhtml

<meta name="viewport" content="width=device-width, initial-scale=1.0” />

This meta element tells your browser to use the whole width of the window or screen for the webpage.

<title></title>

This meta element provides the title of your website. It may get displayed in the titlebar of your window, or in the tab-name if you are using chrome. It usually gets displayed somewhere by the browser and is not optional.

<link rel="stylesheet" href="mycssfile.css">

You will also place links to any stylesheets (ie. CSS files) you are using in the header of your html document. If you are importing javascripts from elsewhere, you will place the url for them in a similar link element.

The Body

All of the content in your page goes under "body", including the part you might think of as resembling a header on a newspaper.

<body></body>

This element is wrapped around all the text, pictures, links, scripts, and everything else in your body.

<p></p>

Separates the enclosed content from the surroundings by a carriage return. This is how you mark up a paragraph.

<em></em>

This means the enclosed text is emphasized. Use it for text you might think of as being spoken loudly or slowly. Different machines will interpret this differently.

<div></div>

<span></span>

These don't have any inherent meaning. They are usually used along with a "class" or "id" attribute to mark a section of text so that something special can be done to it. So, for example, if you wanted a particular block of text to have a particular style, you can use <div class="mystyle"></div> so that you can style the block "mystyle" specially in your CSS. You can do similar things with Javascript, such as using the class name to hide a block of text inside the aforementioned div until the user clicks a button.

<h1></h1>

Use this for the title of your page.

<h2></h2><h3></h3> etc...

Use these to mark chapter headings, subtitles, and subsections of the text on your page.

<ul></ul>

An unordered list. This is not only the stuff you'd put bullet points next to in a presentation, but less obvious things, like menus and nav bars.

<ol></ol>

An ordered list. Both the things you'd want to put numbered items next to in a presentation, but anything that needs to be in a specific order. You can reverse the order by using the attribute "reversed", like <ol reversed></ol>

<li></li>

These are enclosed in ol or ul tags, and are wrapped around items in a the list. Example:

<ol>

<li> This is a list item. </li>

<li> This is also a list item. </li>

</ol>

<dl></dl>

This is used to wrap a description list consisting of terms and defintions. Instead of using <li></li> in your list, use pairs of <dt>This is a term</dt> and <dd>This is a definition.</dd>

<a href="https://link" target="_blank"> This is a link. </a>

The "a" element, which stands for "anchor" is usually used with the href attribute for links. It can be used also with the target attribute to tell the browser where to open the new link. Providing the value "_blank" to this attribute will open the link in a new tab. Instead of using a URL for the value of HREF, you can put an id attribute on any tag and link to that id attribute like this:

<a href="#Chapter_3"> Chapter 3 </a>

...

<h3 id="Chapter_3"> Chapter 3 </h3>

This can be a good way to build a table of contents or to link to a specific part of a page.

<img src="image.jpg" alt="This is alt text for this image." />

This is how you insert an image in the page. The alt text should be descriptive. Here are some hints on how to write good, accessible alt-text. https://webaim.org/techniques/alttext/

<em></em>

This tag means that the enclosed text ought to be emphasized. Use it to draw attention to text that is not necessarily more imporant than the surrounding text. Most browsers will put it in italics unless told otherwise by CSS.

<strong></strong>

Use this to indicate text that is more important than the surrounding text. Most browsers will bold it unless told otherwise by CSS.
